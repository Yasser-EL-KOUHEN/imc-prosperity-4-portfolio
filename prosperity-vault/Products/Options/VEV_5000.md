## Why Bid-Only (One-Sided) Quoting

**Key research finding:** 94–98% of observed trades on VEV_5000 and VEV_5200 **hit the bid** — meaning almost all counterparties want to sell, not buy. Posting asks into this flow would result in immediate adverse selection (we sell, market takes, price moves up → loss).

```python
bid_only = True  # Hard constraint — NO ask orders ever generated
```

The code uses multiple guards:
```python
if config["bid_only"]:
    return []  # skip ask-order generation entirely
# Plus: delta-bypass conditions are also blocked (OPT-05 strictness)
```

**Decision rationale:** Even if our options delta exposure would normally motivate an ask (e.g., long delta needs to sell options), we override and forbid it for 5000/5200. The asymmetric flow evidence is too strong.
